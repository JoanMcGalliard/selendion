Suite without class

We can html files and directories to a suite. If a directory is added, then all the html files in that directory will become part of the suite. When we run a suite, for each html file it looks for a class that has the same name + "Test". If that exists, it is run, otherwise it looks for a class with the same name as html file and runs that. Finally, if neither is found, it creates a class file, based on the current class, for the test. If the class is found, it must be a subclass of SelendionTestCase.

Test that has no class associated with it

If we run a suite containing a test that has no class file, one should be created and the test run: 

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClass.html">without class</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> thread.</p>

The result should be:

<p>Build a suite consisting of
    <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClass.html">without class</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> <ol> <li>
    <a href="suite/WithoutClass.html" class="success">Without class</a> </li> </ol> thread.</p>

Failing test that has no class associated with it

If we run a suite containing a test that will fail and has no class file, one should be created and the test run: 

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClassFail.html">Test without class that fails</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> thread.</p>

The result should be:

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClassFail.html">Test without class that fails</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> <ol> <li> <a href="suite/WithoutClassFail.html" class="failure">Test that fails</a>
    Specification has failure(s). See output HTML for details.</li> </ol> thread.</p>

Test that throws an exception and has no class associated with it

If we run a suite containing a test that will throw and exception and has no class file, one should be created and the test run and the exception should be shown: 

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClassError.html">Test without class that throws an exception</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> thread.</p>

The result should be:

<p>Build a suite consisting of
    <a concordion:addToSuite="'suite',#HREF" href="suite/WithoutClassError.html">Test without class that throws an exception</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> <ol> <li>
    <a href="suite/WithoutClassError.html" class="failure">Test without class that throws an exception</a>
    Unknown command blahdeblah in namespace http://www.selendion.org/2008.</li> </ol> thread.</p>
