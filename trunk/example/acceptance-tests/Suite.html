Suite

We can html files and directories to a suite. If a directory is added, then all the html files in that directory will become part of the suite. When we run a suite, for each html file it looks for a class that has the same name + "Test". If that exists, it is run, otherwise it looks for a class with the same name as html file and runs that. Finally, if neither is found, it creates a SelendionTestCase for the test. If the class is found, it must be a subclass of SelendionTestCase.

Test runs and passes with literal number of threads

If we run a suite containing a simple test that has a test class and it passes: 

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/Sleep.html">Sleep Test</a>
   and run it in <i concordion:runSuite="'suite',1">1</i> thread.</p>

The result should be:

    <p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/Sleep.html">Sleep Test</a>
       and run it in <i concordion:runSuite="'suite',1">1</i> <ol>
    <li> <a href="suite/Sleep.html" class="success">Sleep Test</a> </li> </ol> thread.</p>

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/Concordion.html">Test using ConcordionTestCase</a>
   and run it in <i concordion:runSuite="'suite',#TEXT">1</i> thread.</p>

The result should be:

<p>Build a suite consisting of <a concordion:addToSuite="'suite',#HREF" href="suite/Concordion.html">Test using ConcordionTestCase</a>
       and run it in <i concordion:runSuite="'suite',#TEXT">1</i> <ol> <li>
    <a href="suite/Concordion.html" class="failure">Test using ConcordionTestCase</a>
    class commands.suite.Concordion is not a SelendionTestCase.</li> </ol> thread.</p>
