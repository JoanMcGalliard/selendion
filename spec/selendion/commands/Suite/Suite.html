Suite

You can run Selendion tests in suites by adding then to a suite and subsequently running that suite.

We can add html files and directories to a suite. If a directory is added, then all the html files in that directory will become part of the suite. When we run a suite, for each html file it looks for a class that has the same name + "Test". If that exists, it is run, otherwise it looks for a class with the same name as html file and runs that. If the class is found, it must be a subclass of SelendionTestCase. Finally, if neither is found, it creates a class, based on the current class, for the test.

addToSuite

To add to a suite, this is the syntax:

concordion:addToSuite="suiteName,path"

where suiteName is the the name of the suite (you can create multiple suites and run them separately), and path is the name of the file or directory you want add to the suite. Both can be literals, variables or special variables. The suite name should be a string and the path should be a relative path to a file or a directory, or an absolute path to one within this package. Suites do not have to be initialized, if a new name is used, a new suite is automatically started.

Examples

The following should all successfully added to suite:

  • Add <i concordion:addToSuite="'mySuite','suite/Sleep.html'">Sleep Test</i> to the suite.
  • Add <a concordion:addToSuite="'mySuite',#HREF" href="suite/Sleep.html">Sleep Test</a> to the suite.
  • Add to <i concordion:set="#suiteName">mySuite</i>: <a concordion:addToSuite="#suiteName,#HREF" href="suite/Sleep.html">Sleep Test</a>.

runSuite

To run a suite, this is the syntax:

where suiteName is the the name of the suite, and numberOfThreads is the name of the file or directory you want add to the suite. Both can be literals, variables or special variables. The suite name should be a string, and will be a suite created by one or more addToSuites and the numberOfThreads should be an integer. If numberOfThreads is 1, then the tests in the suite will be run one after the other. If it is greater than 1, then it will fork up to that many processes to run each test in parallell. Suites do not have to be initialized, if a new name is used, nothing should be run.

Example

A suite with a single, passing test.

If we add one test that will pass to a suite: Add <a concordion:addToSuite="'suite1',#HREF" href="suite/Sleep.html">Sleep Test</a> to the suite.

And then run that suite:

<p>run it in <i concordion:runSuite="'suite1',1">1</i> thread.</p>

Then we would expect the test to report success:

<p>run it in <i concordion:runSuite="'suite1',1">1</i>
     <ol> <li> <a href="suite/Sleep.html" class="success">Sleep Test</a> </li> </ol>
     thread.</p>

Example

An empty suite

If we try to run a suite without adding any tests

<p>run it in <i concordion:runSuite="'nonExistentSuite',1">1</i> thread.</p>

Then we would expect the test to report success, with no tests executed:

<p>run it in <i concordion:runSuite="'nonExistentSuite',1">1</i> <ol /> thread.</p>

Example

Test is not a Selendion test

If create a suite containing a test class that is not subclassed from SelendionTestCase:

Build a suite consisting of <a concordion:addToSuite="'suite3',#HREF" href="suite/Concordion.html">Test using ConcordionTestCase</a>

And then run that suite:

Run it in <i concordion:runSuite="'suite3',#TEXT">1</i> thread.

The result should be:

Run it in <i concordion:runSuite="'suite3',#TEXT">1</i> <ol> <li>
    <a href="suite/Concordion.html" class="failure">Test using ConcordionTestCase</a>
    class selendion.commands.Suite.suite.Concordion is not a SelendionTestCase.</li> </ol> thread.

Example
